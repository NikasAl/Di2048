# Скриншоты для RuStore и Google Play

## Требования

- **Формат:** PNG
- **Разрешение:** 1080×1920 (9:16) — обязательно для RuStore
- **Минимум:** 5 скриншотов
- **Рекомендуется:** 8 скриншотов

## Сценарии скриншотов (обязательные)

| № | Сценарий | Описание | Подпись для RuStore |
|---|----------|----------|---------------------|
| 1 | `main_menu_4x4` | Главное меню / игровое поле 4×4 | «Классическая головоломка 2048» |
| 2 | `field_8x8` | Поле 8×8, показать масштаб и рекорд | «10 размеров поля: от 3×3 до 12×12» |
| 3 | `game_over` | Экран Game Over с кнопками «Продолжить» и «Отменить» | «Продолжи игру после проигрыша» |
| 4 | `undo_button` | Кнопка UNDO в статической панели (крупно) | «Отмени последний ход» |
| 5 | `settings` | Настройки — выбор размера поля | «Выбери сложность под себя» |

## Дополнительные сценарии (рекомендуемые)

| № | Сценарий | Описание | Подпись |
|---|----------|----------|---------|
| 6 | `field_12x12` | Максимальное поле 12×12 | «Бесконечная игра на 12×12» |
| 7 | `dark_theme` | Тёмная тема в действии | «Современный тёмный дизайн» |
| 8 | `high_score` | Экран с высоким рекордом | «Побей свой рекорд!» |

## Автоматический скрипт

Используйте интерактивный скрипт для создания скриншотов:

```bash
./scripts/screenshots.sh
```

### Что делает скрипт:

1. ✅ Проверяет подключение устройства через ADB
2. ✅ Предлагает установить разрешение 1080×1920 (9:16)
3. ✅ Позволяет выбирать сценарии скриншотов
4. ✅ Сохраняет файлы с правильными именами: `scr_N_scenario.png`
5. ✅ Автоматически восстанавливает исходное разрешение при выходе

### Пошаговый процесс:

1. **Подключите устройство** через USB с включённой отладкой
2. **Запустите скрипт:**
   ```bash
   ./scripts/screenshots.sh
   ```
3. **Ответьте Y** на предложение установить разрешение 9:16
4. **Запустите приложение** на устройстве
5. **Выбирайте сценарии** по очереди (1-6):
   - Скрипт подскажет, какой экран нужен
   - Перейдите к этому экрану в приложении
   - Нажмите Enter — скриншот сохранится
6. **Повторите** для всех 5+ сценариев
7. **Нажмите R** для выхода и восстановления разрешения

## Ручной режим (альтернатива)

Если скрипт не работает, можно делать скриншоты вручную:

```bash
# Установить разрешение 9:16
adb shell wm size 1080x1920

# Сделать скриншот
adb shell screencap -p /sdcard/screenshot.png
adb pull /sdcard/screenshot.png Materials/screens/scr_1_manual.png

# Восстановить разрешение
adb shell wm size reset
adb shell wm density reset
```

## Проверка скриншотов

Перед загрузкой проверьте:

```bash
# Проверить разрешение всех скриншотов
for f in Materials/screens/*.png; do
    echo "$f: $(file "$f")"
done

# Или через identify (требуется ImageMagick)
identify Materials/screens/*.png
```

Все скриншоты должны быть **1080×1920 PNG**.

## Именование файлов

Формат: `scr_N_scenario.png`

- `N` — порядковый номер (1, 2, 3...)
- `scenario` — краткое описание сценария (main_menu_4x4, field_8x8, etc.)

Примеры:
- `scr_1_main_menu_4x4.png`
- `scr_2_field_8x8.png`
- `scr_3_game_over.png`
